# SetFlow — Концепция: AI-ассистент руководителя

## Контекст

Проблема: руководитель съёмочной группы перегружен рутиной. Он одновременно координатор, логист, технический директор и коммуникатор. SetFlow не должен добавлять ему работы ("ещё одна система для заполнения"), а должен **снимать нагрузку**. Ключевой механизм — AI-агент, который живёт внутри приложения.

## Боли руководителя (что решаем)

| Боль | Как сейчас | Как в SetFlow |
|------|-----------|---------------|
| Координация команды | WhatsApp-чаты, голосом, забывается | Задачи с дедлайнами + чат внутри проекта |
| "Все ли взяли?" | Звонки утром, нервы | Чеклисты с подтверждением, AI напоминает |
| Расстановка на площадке | В голове или на бумажке | 2D-план + шаблоны + AI-подсказки |
| Что снять, что забыли | Опыт + shot list на бумаге | AI анализирует shot log vs сценарий |
| Клиент спрашивает статус | Звонки, переписка | Клиентский портал — сам видит |
| Подготовка call sheet | Excel + Word, 30 минут | Авто-генерация PDF по данным проекта |
| Финансы | Блокнот, Excel | Бюджет в проекте, факт/план |

## Ключевой принцип: AI-агент внутри

**SetFlow AI** — не просто подключение к моделям, а **встроенный ассистент**, который:

### 1. Проактивно помогает (не ждёт команды)
- Утром в день съёмки: автоматически генерирует call sheet, проверяет погоду, напоминает команде о времени сбора
- Видит незакрытые чеклисты — пингует ответственных
- Анализирует shot log в реальном времени: "Сцена 3 не снята, осталось 40 минут до wrap"
- После съёмки: "Загружено 148 файлов, 12.4 ГБ. Создать альбомы по сценам?"

### 2. Генерирует контент
- Сценарий → раскадровка → план площадки (черновик)
- Описание проекта → коммерческое предложение для клиента
- Shot log → highlight-лист для монтажёра
- Интервью (аудио) → транскрипция → субтитры

### 3. Анализирует и советует
- "Для интервью в помещении обычно используют 3 источника света. У вас назначено 2. Добавить?"
- "Оператор Орлов был занят на 3 из последних 4 проектов. Доступность под вопросом."
- "Бюджет на аренду превышен на 15%. Показать альтернативы?"

### 4. Учится на шаблонах и истории
- Шаблоны съёмок (интервью, свадьба, репортаж...) → AI знает типовую расстановку, оборудование, команду
- История прошлых проектов → "В прошлый раз на свадьбе в этом зале использовали 2 Aputure + софтбокс"

## Новые модули (что добавить)

### М1: Трекер задач (Task Board)
**Зачем:** Руководитель раздаёт задания, контролирует выполнение.

- Канбан-доска внутри проекта: Новое → В работе → Готово
- Задачи привязаны к: съёмочному дню, сцене, члену команды
- Приоритеты, дедлайны, вложения
- Переназначение задач (делегирование)
- AI: автоматически создаёт задачи из сценария ("Проверить петличку перед интервью" → задача на звукорежиссёра)
- Уведомления: push/telegram/email

**Примеры задач:**
- "Зарядить 6 батарей для FX6" → Ассистент → дедлайн: вечер перед съёмкой
- "Проверить разрешение на съёмку дроном" → Продюсер → дедлайн: за 3 дня
- "Привезти отражатель 5-в-1" → Осветитель → дедлайн: день съёмки, 07:00

### М2: Чат проекта
**Зачем:** Вся коммуникация внутри SetFlow, не в WhatsApp.

- Общий чат проекта
- Чаты по съёмочным дням ("Церемония — 15 апреля")
- Личные сообщения внутри проекта
- Вложения: фото, файлы, ссылки на записи базы знаний
- AI в чате: можно спросить "@setflow какая погода на завтра?", "@setflow сгенерируй call sheet"
- Чат с клиентом (через портал)

### М3: Шаблоны съёмок (Production Templates)
**Зачем:** Не начинать каждый проект с нуля.

Шаблон включает:
- Типовой состав команды и роли
- Типовой набор оборудования (kit preset)
- Типовой сценарий / shot list
- Типовой план площадки (2D)
- Типовые чеклисты по ролям
- Типовые задачи
- База знаний шаблона (советы, лайфхаки, частые ошибки)

**Примеры шаблонов:**
| Шаблон | Команда | Оборудование | Особенности |
|--------|---------|-------------|-------------|
| Интервью в помещении | 3 чел. (оператор, звук, осветитель) | 2 камеры, 3 света, boom + петличка | 2-камерная схема, 3-точечный свет |
| Интервью на улице | 2 чел. (оператор, звук) | 1 камера, отражатель, беспроводной звук | ND фильтры, ветрозащита |
| Свадьба (полный день) | 5 чел. | 3 камеры, дрон, 4 света, полный звук | 3 блока, 12+ сцен |
| Репортаж / событие | 2 чел. | 1 камера, накамерный свет, петличка | Мобильность, быстрая реакция |
| Корпоративный ролик | 4 чел. | 2 камеры, свет, звук, телепромтер | Сценарий, дубли, постановка |
| Документалка (выезд) | 3-5 чел. | По экспедиции: защита, батареи, рация | Автономность, погодные условия |

Создание проекта: "Выбери шаблон → скорректируй → начни работу". AI подсказывает: "Для зала 120м² рекомендую добавить ещё один источник заполняющего света."

### М4: AI-ассистент SetFlow (панель справа + чат)

**Архитектура AI:**
```
Пользователь
     ↓
  SetFlow AI (оркестратор)
     ├── Контекст проекта (сценарий, команда, оборудование, расписание)
     ├── Шаблоны и база знаний
     ├── История действий (shot log, задачи, чат)
     └── Подключённые модели (Claude, Whisper, Topaz...)
          ↓
      Ответ / Действие / Рекомендация
```

**Где живёт AI:**
- **Правая панель** — контекстные подсказки, рекомендации, алерты
- **Чат** — можно спросить что угодно о проекте
- **Автоматические действия** — фоновые задачи (транскрипция, генерация PDF, анализ)

**Примеры сценариев AI:**

**Сценарий 1: Подготовка к съёмке**
> Руководитель создал проект "Интервью: Автор Петров" из шаблона "Интервью в помещении".
> AI: "Шаблон рекомендует 3 источника света. Добавил Aputure 600d, Godox SL-200, кольцевую. Оператор Волков свободен на эту дату. Назначить?"
> Руководитель: "Да"
> AI генерирует задачи: "Зарядить батареи" (ассистент), "Проверить петличку" (звук), "Подготовить вопросы" (руководитель)

**Сценарий 2: На площадке**
> Идёт съёмка интервью. AI в реальном времени:
> - "Снято 45 минут из запланированных 60. 3 вопроса не заданы."
> - "Батарея камеры A — 23%. Запасная у ассистента, задача отправлена."
> - "Звук: уровень упал на канале 2. Возможно, батарея петлички."

**Сценарий 3: После съёмки**
> AI: "Загружено 234 файла (45 ГБ). Создал альбомы: «Интервью дубли» (12 видео), «B-roll» (8 видео), «Фото» (214 фото)."
> AI: "Транскрипция интервью готова (Whisper). 12 страниц. Хотите выделить ключевые моменты?"
> AI: "Клиентский портал обновлён. Отправить ссылку автору?"

## Что менять в текущем проекте

### Новые маршруты
```
/projects/[id]/tasks        — Трекер задач (канбан)
/projects/[id]/chat         — Чат проекта
/projects/[id]/ai           — AI-подключения (уже есть) + AI-ассистент
/templates                  — Шаблоны съёмок (org-level)
/templates/[id]             — Редактор шаблона
```

### Новые таблицы в схеме
```
project_tasks         — задачи проекта (title, assignee, status, priority, dueDate, shootDay, scene)
task_comments         — комментарии к задачам
chat_messages         — сообщения чата (project, channel, author, text, attachments)
chat_channels         — каналы чата (project, type: general/shoot_day/dm, name)
production_templates  — шаблоны съёмок (name, type, description, data: JSONB)
ai_conversations      — история диалогов с AI (project, messages[], context)
ai_actions            — автоматические действия AI (project, type, status, result)
```

### Изменения в nav
```
Org-level:
  + Шаблоны (в секции ХРАНИЛИЩЕ)

Project workspace:
  КОММУНИКАЦИЯ (новая секция):
    + Задачи
    + Чат
  УПРАВЛЕНИЕ:
    AI-подключения → AI-ассистент (расширенное)
```

### Изменения в правой панели проекта
Сейчас: Активность, Заметки, Комментарии
Станет: **AI-ассистент** — контекстные рекомендации, алерты, быстрые действия

## Приоритет реализации

1. **Трекер задач** — ключевая боль, канбан с привязкой к съёмочным дням
2. **Шаблоны** — ускорение создания проектов
3. **AI-ассистент (правая панель)** — контекстные подсказки
4. **Чат проекта** — замена WhatsApp
5. **AI-автоматизации** — фоновые действия
